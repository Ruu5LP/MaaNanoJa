# 仕様（SPEC）

このアプリの「振る舞い」を明文化したもの。コードを読む前にここを読めば、何をどう計算しているかが分かる。
実装の正典は `src/lib/`（純粋ロジック）と、それを固定している `src/lib/scoring.test.ts`（実行できる仕様）。

## 対象

日本式リーチ麻雀・4人打ち・半荘（東南戦）。月イチの家麻雀のスコアと局データを記録する。

## スコア計算（半荘の最終持ち点 → 順位・スコア）

既存の [namimori 氏の麻雀集計スプレッドシート](https://hirokuasaku-live.blogspot.com/2022/08/mahjong-spreadsheet-new.html) と
**完全一致**させる（`scoring.test.ts` が過去9試合で検証）。

- スコアの単位は「点数 ÷ 1000」（例: +30.0）。
- 各プレイヤーのスコア:
  ```
  score = (最終持ち点 - 返し点) / 1000 + uma[順位] + (1位なら oka)
  oka   = (返し点 - 配給原点) × 人数 / 1000     ← 1位にのみ加算
  ```
- 既定ルール: 配給原点 25000 / 返し点 30000 / ウマ `[+30, +10, -10, -30]`（10-30）。
  → oka = (30000-25000)×4/1000 = **20**。設定画面で変更可能。
- **ゼロサム**: 全員のスコア合計は常に 0（オカが素点合計のマイナス分を埋める）。
- **同点の扱い**: 上家優先（`tiebreak: 'shimocha'`）。点数が同じなら席順（起家に近い= `seatIndex` が小さい）が上位。

### 検証済みの例（game1）

| プレイヤー | 最終持ち点 | 順位 | スコア |
| ---------- | ---------: | :--: | -----: |
| もっそ     |     57,600 |  1   |  +77.6 |
| れいに     |     20,900 |  2   |   +0.9 |
| さーりゃん |     14,300 |  3   |  -25.7 |
| 超髪       |      7,200 |  4   |  -52.8 |

## 局ごとの点数移動（局ログ → 持ち点）

局ログを入れると持ち点が自動で動く。実装は `scoring.ts#handDeltas` と `game.ts#replay`。

### 和了点（翻・符 → 点）

基本点 `base = fu × 2^(2+han)`（満貫以上は翻数で固定: 満貫2000 / 跳満3000 / 倍満4000 / 三倍満6000 / 役満8000）。

- 子ロン `= ceil100(base×4)` / 親ロン `= ceil100(base×6)`
- 子ツモ: 子 `ceil100(base)` ×2 + 親 `ceil100(base×2)`
- 親ツモ: 子 `ceil100(base×2)` ×3
- `ceil100` は100点単位の切り上げ。

### 供託・本場

- **立直棒**: 立直者は -1000 して場（pot）に積む。和了者が pot を総取り。流局時は pot を場に残す（次局へ持ち越し）。
- **本場**: ロンは放銃者が +300×本場、ツモは各家が +100×本場を追加で払う。

### 流局

- テンパイ n 人 / ノーテン (4-n) 人で 3000 点を移動（テンパイ +3000/n、ノーテン -3000/(4-n)）。
- 途中流局（abortive）は点数移動なし・供託据え置き。

### 親の連荘・本場・局送り（`game.ts#advance`）

- 親の和了、または流局時に親テンパイ → **連荘**（親そのまま・本場+1）。途中流局も連荘。
- それ以外 → 親流れ（次の席が親・本場0）。東4の次は南1。

## データモデル

型定義がそのまま仕様: `src/lib/domain.ts` を参照。要点だけ:

- `DB` = `{ version, players, rules, games }`。localStorage キー `mahjong-tracker/v1` に保存。
- `Game.playerIds` は席順（起家=index0, 東南西北）。
- `Game.hands` が空なら `finalPoints`（手入力）を最終持ち点として使う。局ログがあれば `replay` の結果を使う。
- `Hand` は **判別可能ユニオン**（`type` で分岐）。ロンには `winner/loser/han/fu`、流局には `tenpai`、が型で保証される。

## 不変条件（壊してはいけないもの）

1. スコア計算は既存スプレッドシートと一致し続ける（`scoring.test.ts` が守る）。変更時は必ずテストを通す。
2. 半荘スコアはゼロサム。
3. 保存データは後方互換に読める（`normalizeDB` が未知の形を吸収）。スキーマを変えるときは `version` と移行を用意する。
